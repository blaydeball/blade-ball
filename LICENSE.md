## 🔧 Common Issues & Fixes

### ❌ "Auto Parry isn't triggering — balls pass through without deflection"
**Fix:** This almost always means the script is running an outdated build after a *Blade Ball* game update. The parry hook relies on game-specific signal paths that change with patches. Download the latest version from the **blade ball script github** repository. If you're already on the latest version, check the `#patch-status` channel on the Discord for an active hotfix.

---

### ❌ "GUI doesn't appear after execution"
**Fix:** Your executor may not support the DrawingLib or UI library used by the script. Switch to a fully UNC-compliant executor (Synapse X, Zorara, Velocity, or KRNL). Also confirm that you executed the script *after* injecting — executing before injection will silently fail.

---

### ❌ "Kill All keeps targeting already-eliminated players"
**Fix:** This was a known targeting cache bug fixed in v5.23. Update to the latest version. If you are already on v5.23, toggle Kill All off and back on to force a target list refresh.

---

### ❌ "Script works in solo rounds but breaks in team modes"
**Fix:** Team mode uses a different player filtering system. Open the GUI **Settings** tab and enable **Team Mode Compatibility**. This switches the kill targeting logic to exclude teammates from the parry redirect pool.

---

### ❌ "Mobile GUI is cut off / partially off-screen"
**Fix:** Use the **Mobile Build** download specifically — the standard PC build's GUI is not auto-scaled for mobile screens. The mobile build includes a resolution-aware layout system that adjusts to your device's screen dimensions automatically.

---

### ❌ "HttpGet fails when using the loadstring method"
**Fix:** Enable HTTP requests in your executor's settings. If your executor doesn't support `HttpGet`, use the direct file download method and paste the script content manually into the editor instead.

---

### ❌ "Parry timing feels inconsistent on high ping"
**Fix:** Open the GUI **Settings** tab and enable **Predictive Parry Mode**. This mode compensates for server latency by triggering the parry action slightly earlier than the raw detection signal, producing more consistent results on connections above 100ms.
